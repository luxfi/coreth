# JSON-RPC API

The C-Chain exposes a JSON-RPC API compatible with Ethereum tooling. All standard `eth_*` methods are supported along with additional Lux-specific methods.

## Endpoints

| Network | HTTP Endpoint | WebSocket |
|---------|--------------|-----------|
| Mainnet | `https://api.lux.network/ext/bc/C/rpc` | `wss://api.lux.network/ext/bc/C/ws` |
| Testnet | `https://api.testnet.lux.network/ext/bc/C/rpc` | `wss://api.testnet.lux.network/ext/bc/C/ws` |
| Local | `http://localhost:9650/ext/bc/C/rpc` | `ws://localhost:9650/ext/bc/C/ws` |

## eth_* Namespace

### eth_chainId

Returns the chain ID.

```bash
curl -X POST -H "Content-Type: application/json" --data '{
  "jsonrpc": "2.0",
  "method": "eth_chainId",
  "params": [],
  "id": 1
}' https://api.lux.network/ext/bc/C/rpc
```

Response:
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": "0xa86a"
}
```

### eth_blockNumber

```bash
curl -X POST -H "Content-Type: application/json" --data '{
```

### eth_getBalance

Returns the balance of an address.

**Parameters:**
1. `address` - Address to check
2. `block` - Block number or "latest", "earliest", "pending"

```bash
curl -X POST -H "Content-Type: application/json" --data '{
  "jsonrpc": "2.0",
  "method": "eth_getBalance",
  "params": ["0x...", "latest"],
  "id": 1
}' https://api.lux.network/ext/bc/C/rpc
```

### eth_call

Executes a call without creating a transaction.

**Parameters:**
1. `transaction` - Transaction object
2. `block` - Block number or tag

```bash
curl -X POST -H "Content-Type: application/json" --data '{
  "jsonrpc": "2.0",
  "method": "eth_call",
  "params": [{
    "to": "0x...",
    "data": "0x..."
  }, "latest"],
  "id": 1
}' https://api.lux.network/ext/bc/C/rpc
```

### eth_estimateGas

### eth_feeHistory

Returns historical fee data (EIP-1559).

**Parameters:**
1. `blockCount` - Number of blocks
2. `newestBlock` - Newest block number or tag
3. `rewardPercentiles` - Percentiles to report

```bash

  "id": 1

```

```bash
    "address": "0x...",
    "topics": ["0x..."]
  }],
  "id": 1
}' https://api.lux.network/ext/bc/C/rpc
```

## Subscription Methods (WebSocket)

### eth_subscribe

Subscribe to events.

**Types:**
- `newHeads` - New block headers
- `logs` - Log events
- `newPendingTransactions` - Pending transactions

```javascript
const ws = new WebSocket('wss://api.lux.network/ext/bc/C/ws');

ws.send(JSON.stringify({
  jsonrpc: "2.0",
  method: "eth_subscribe",
  params: ["newHeads"],
  id: 1
}));
```

### eth_unsubscribe

Unsubscribe from events.

## debug_* Namespace

Requires explicit enablement in node configuration.

### debug_traceTransaction

Traces transaction execution.

**Parameters:**
1. `txHash` - Transaction hash
2. `options` - Tracer options

```bash
curl -X POST -H "Content-Type: application/json" --data '{
  "jsonrpc": "2.0",
  "method": "debug_traceTransaction",
  "params": ["0x...", {"tracer": "callTracer"}],
  "id": 1
}' https://api.lux.network/ext/bc/C/rpc
```

### debug_traceBlockByNumber

Traces all transactions in a block.

### debug_traceCall

Traces a call without executing.

## txpool_* Namespace

### txpool_status

### txpool_inspect

## Batch Requests

Multiple requests can be batched:

```bash
curl -X POST -H "Content-Type: application/json" --data '[
  {"jsonrpc": "2.0", "method": "eth_chainId", "params": [], "id": 1},
  {"jsonrpc": "2.0", "method": "eth_blockNumber", "params": [], "id": 2}
]' https://api.lux.network/ext/bc/C/rpc
```

## Rate Limits

Public endpoints have rate limits:

| Tier | Requests/sec | Burst |
|------|--------------|-------|
| Free | 10 | 50 |
| Standard | 100 | 500 |
| Enterprise | Custom | Custom |

For higher limits, run your own node or contact Lux for enterprise access.
